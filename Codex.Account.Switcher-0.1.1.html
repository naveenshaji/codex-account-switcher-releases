Codex Account Switcher 0.1.1

OAuth login reliability update.

Fixed

  • Fixed Add Account failures caused by GUI launch environments that could find codex but not its node runtime
  • Improved OAuth diagnostics so app-server exit status and stderr are surfaced instead of a generic close message
  • Improved auth-file recovery after login completion in temporary CODEX_HOME environments

Notes

  • If you previously saw status 127 during Add Account, this is the fix for that issue
  • Running Codex desktop sessions still require restart after account switches
  • CLI sessions still require opening a new session after account switches
